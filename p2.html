Creating Fragments

Designing your app with modularity in mind suggests that you use fragments as the building blocks for your application rather than activities alone. Our goal, then, will be to start building our application using fragments. There are some basic steps involved:

  • Create a Java class that extends Fragment
  • Create an XML layout file defining the appearance of that fragment
  • Add the fragment to the activity
  • Inflate the fragment

Let's go ahead and get started with our first fragment app. Fire up Android studio (I'm using version 2.2) and set up a New Project:

Click next and then accept the default values. For the minimum SDK, API 15: Android 4.0.3 (IceCreamSandwich) will work just fine. Click Next again, and select the Master/Detail Flow activity.

Once ANdroid Studio (AS) has generated the code, you can drill in to see how this is set up. In the java/com.example.admin.fragmentsdemo/dummy directory, you'll see a dummy set of Item* files. One file is the ItemDetailActivity, one is the ItemDetailFragment, and one is the ItemListActivity. The ItemDetailActivity exists to display the details of an item once clicked upon, while the Fragment is the reusable component. ItemListActivity is an activity used to display a list of items.

The ItemDetailFragment class is extending Fragment. The code generated some dummy values and methods for a developer to work with. It overrides the onCreate and onCreateView methods. We will deal with these details more, but if you read the official android guide, these methods should be familiar to you.

What's Happening

There is a bit of logic going on that the code generation took care of us. In the ItemListActivity file, the following code is setting a boolean value depending on the size of device being used: 

if (findViewById(R.id.item_detail_container) != null) {
    // The detail container view will be present only in the
    // large-screen layouts (res/values-w900dp).
    // If this view is present, then the
    // activity should be in two-pane mode.
    mTwoPane = true;
}

We will revisit this later. In the next segment, we are going to start a new project (again) and build our own fragment from scratch so that we can understand how to implement this logic on our own.
