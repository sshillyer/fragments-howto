Pre-Requisites

The fragments API allows a developer to creature modular components that can be reused throughout an application. Fragments have their own lifecycle similar to an Activity. However, an Activity can itself contain one or more fragments.

Why does this matter? In a nutshell, it allows you to design reusable components that can be combined in different ways on devices with varying sizes. The rest of this article assumes that you've read the official android developer guide for fragments. The guide does provide a basic overview and a few code snippets, but it can be hard to see how the pieces fit together. This guide strives to get the novice android developer comfortable with fragments in a practical way.

In addition, it is assumed that you are familiar with the basics of Android apps including the lifecycle of an Activity, resource files (including those in res/layout and res/values), and how to navigate within the Android Studio environment.

This guide is best used as a practical, step-by-step tutorial rather than a reference guide, giving you hands-on experience rather than a textual explanation of the API. When appropriate, links to the official android reference material are provided so that you can dive deeper into the details that you find most interesting.

Before Continuing

If you do not have a tablet and a phone to test the applications on during this tutorial, make sure to install a few devices (I'm using Nexus 6 and a Nexus 10) using the ADK in Android Studio. For help, go to the official developer page.
